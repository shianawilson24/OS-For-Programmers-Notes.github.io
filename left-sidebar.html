Units 1 - 3

Vocab

  • Basics of a computer system: CPU, RAM, secondary storage, I/O devices
  • OS (Operating System): Software that runs on the bare hardware of a computer and provides essential support for users to develop and use applications in the most efficient and safe manner.
  • CPU: Machine instructions perform operations on contents of registers and memory locations
  • Main memory: Physical memory is a linear sequence of addressable bytes or words that hold programs and data
  • Secondary storage: Disk and other secondary storage devices are multi-dimensional structures which require complex sequences of low-level operations to store and access data organized in discrete blocks
  • I/O devices: Read and write registers of the device controllers
  • Multiprogramming: Technique that keeps several programs active in memory and switches execution among the different programs to maximize the use of the CPU and other resources
  • Time sharing (multitasking): Extension of multiprogramming where the CPU is switched periodically among all active computations to guarantee acceptable response times to each user. It’s related to visualization by creating the illusion of having separate virtual CPUs for each computation.
  • Kernel: The OS is the minimal set of functions necessary to manage the system resources safely and efficiently
  • Privileged instruction: Performs critical operations that access I/O devices and the CPU’s status and control registers. Only the OS kernel is allowed to execute privileged instructions.
  • Kernel mode: CPU state where both privileged and non-privileged instructions may be used
  • User mode: CPU state where only non-privileged instructions may be used
  • Graphic user interface (GUI): Presents various icons on the screen, which the user can click on in different ways to invoke services associated with the icons, or to reveal pull-down menus for additional tasks. (aka a website or application)
  • Shell Commands:
    • DIR: Displays a list of files and subdirectories in a directory folder
    • COPY: Copy one or more files to another location
    • WC: Count the number of words, lines, and characters in a text file
    • DEL: Delete one or more files
  • Shell script: Program that implements a new operation by combining multiple commands and control statements into one named unit interpreted by the shell
  • System call: Request from an application for an OS service. Implemented by a library function that sets up needed parameters for the requested operation and issues a corresponding supervisor call.
  • Supervisor call (kernel call): Privileged instruction that automatically transfers execution control to a well-defined location within the OS kernel. Thus, supervisor calls provide the interface between the OS kernel and the higher-level software
  • Interrupt: Event that diverts the current execution of a program to a predefined location in the kernel in order to respond to an event. Triggered by a hardware signal sent to the CPU from an external device.
  • Trap: AKA internal interrupt. Triggered by the currently executing instruction
  • Interrupt handler: Kernel function invoked whenever an interrupt occurs
  • Moore’s Law: Formulated by Gordon Moore, it is the observation that the number of transistors in an integrated circuit doubles about every two years.
  • Bus: General-purpose computer system that has one or more CPUs and a number of devices connected through a common bus that provides access between components and shared memory
  • Device driver: Understands the device controller and provides the rest of the OS with a uniform interface to the device
  • Network computers (thin clients): Terminals that understand web-based computing are used in place of traditional workstations where more security or easier maintenance is desired
  • Firewalls: Used to protect networks from security breaches
  • Mobile computing: Refers to computing on handheld smartphones and tablet computers
  • Compute server system: Provides an interface to which a client can send a request to perform an action. Example: reading data
  • File-server system: Provides a file system interface where clients can create, update, read, and delete files. Example: web server that delivers files to clients running web browsers
  • Cloud computing: Delivers computing, storage, and even applications as a service across a network
    • Extension of virtualization since it uses it as a base
  • ASICs: Performs tasks without an operating system

Notes

  • Resource utilization is improved by multiprogramming, which overlaps CPU-bound and I/O bound computations. Time sharing only switches the CPU among different computations to create the illusion of simultaneous execution, which generally does not improve resource utilization.
  • Multiprogramming and time-sharing are used together: since it requires multiprogramming
  • To make sure no program operates outside of the trusted kernel can use privileged instructions the CPU operates in two different modes indicated by a special mode bit: Kernel mode and User mode
  • Kernel functions can be invoked by any program
  • The kernel can execute privileged and non-privileged instructions
  • When a user logs in the OS starts a GUI or shell
  • Supervisor call aka kernel call can be executed by any program
  • The first digital computer system developed in 1940 used vacuum tubes to represent 0s and 1s

Five generations of computer systems:

  • Gen 1: Vacuum tubes
    • No OS support
    • All programming was done using machine language
  • Gen 2: Transistors
    • Replaced Vacuum tubes: small and fast switches
    • OS type is batch OS
    • Programming was done in batches of punch cards. Automate compilation, load and execution of programs. Multiprogramming was developed which allows the OS to schedule the execution of jobs to make more efficient use of the CPU and other resources
  • Gen 3: Integration Circuits
    • Allowed the development of microchips to replace individual transistors
    • OS type is Interactive multi-user OS
    • Interrupts were developed to allow the OS to enforce time sharing and to interact with keyboards and displays terminals (which were also developed at the same time). Increased capacity and speed of memory and secondary storage devices imposed additional management tasks on the OS
  • Gen 4: Very Large Scale Integration (VLSI)
    • Allowed placement of a complete microprocessor on a chip leading to the development of PCs
    • OS type is Desktop and laptop OS
    • OS responsible for all operations starting from the initial booting, to multitasking, scheduling, interactions with various peripheral devices, and keeping all information safe. User-friendliness introduced GUI
  • Gen 5: Networking hardware
    • OS type for Supercomputers, distributed systems, and mobile devices
    • Creates powerful chips spawned several directions of development. Supercomputers combined a large number of processors and made the OS and other software responsible for exploiting the increased computation power through parallel processing. Gave rise to the Internet, which imposed requirements of privacy and safety along with efficient communication. Wireless networks led to handheld devices
