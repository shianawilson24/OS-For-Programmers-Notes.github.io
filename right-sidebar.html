Scheduling Concepts


Non preemptive vs preemptive:

Non preemptive scheduling algorithms allow a process to continue until it terminates or gets blocked. While preemptive scheduling algorithms stops the current running process and choose another process whenever and wherever.

  • Long-Term Scheduling: Decides when a process enters the ready list (RL) to compete for CPU time.
  • Short-Term Scheduling: Decides which process runs next on the CPU.
  • Thread Priority: A numerical value indicating the importance of a process relative to others; used by the scheduler to determine which process runs next.
  • Arbitrary Rule: Applied when two or more processes have the same priority to decide which should receive CPU time.
  • Arrival: The point in time when the process enters the ready list (RL).
  • Departure: The point in time when the process leaves the RL by entering a blocked state or by terminating.
  • Real Time in System: The actual time a process has spent in the system since arrival.
  • Total CPU Time: The time a process consumes between arrival and departure; also known as CPU burst.
  • External Priority: A numeric priority value assigned to a process at creation.
  • Batch Process: Performs long-running, repetitive tasks that do not require user intervention.
  • Time Quantum: The smallest amount of time (typically 10 to 100 milliseconds) a process can utilize the CPU.

Scheduling Algorithms


Non-preemptive Scheduling Algorithms

  • First Come First Serve (FCFS): Schedules processes by arrival time; non-preemptive.
  • Shortest Job Next (SJN): Schedules processes by total CPU time required; non-preemptive.

Preemptive Scheduling Algorithms

  • Shortest Remaining Time (SRT): Schedules by remaining CPU time needed; preemptive.
  • Round Robin: Uses a single queue; each process runs for a time quantum (Q) before moving to the back of the queue. Preemptive
  • Multi-Level Scheduling (MLS): Uses different time quantums for each priority level, dynamically changing process priorities to address starvation and fairness.
  • Multi-Level Feedback Queue: Newly arriving processes start at the highest priority; if they exceed the quantum, they move to a lower priority queue with a larger quantum.

Real-Time Scheduling (Also Preemptive)

  • Rate Monotonic Scheduling: Schedules by period; shorter periods have higher priority; preemptive.
  • Earliest Deadline First: Schedules by shortest remaining time until the deadline; preemptive.

Concurrency and Synchronization


  • Concurrency: Multiple processes or threads executing simultaneously; achieved through time-sharing on a single CPU or parallel execution on multiple CPUs.
  • Critical Section: A segment of code that cannot be executed by more than one process at a time.
  • Guaranteeing Mutual Exclusion: Ensures only one process executes within the critical section (CS).
  • Preventing Lockout: Ensures that a process attempting to enter the CS does not prevent others from entering.
  • Preventing Starvation: Ensures that no process is indefinitely denied entry to the CS while others wait.
  • Preventing Deadlock: Avoids scenarios where multiple processes block each other.
  • Semaphore: A primitive for solving synchronization problems.
  • Bounded Buffer Problem: A classic synchronization problem illustrating process cooperation.
  • Busy Waiting: Repeatedly executing a loop while waiting for a condition to change.
  • Monitor: A high-level synchronization primitive using P&V operations, encapsulating data and functions.
  • Resource Allocation Graph: Displays the current allocation of resources to processes and their requests for new resources.
