Vocab

File System Concepts


  • File: A named collection of information managed on secondary storage by the file system. Users can view it as a single abstract unit of data storage.
  • Records: Items identified with a file by a record number or unique key field, e.g., student ID.
  • Access Method: A set of operations provided by the OS to access files. Common methods include sequential and direct access.
  • Metadata: Information about the format and organization of a file's data, generally stored in a file's header.
  • Magic Number: A sequence of characters at the start of the file header identifying the file type.
  • File Extension: A sequence of characters following the filename extension, used to identify the file type.
  • File Directory or Folder: A special-purpose file that records information about other files and directories.
  • Tree Structure Directory Hierarchy: A collection of directories organized such that every directory points to zero or more files or directories.
  • Absolute Path: Uniquely identifies a file by the concatenation of directory and file names leading from the root.
  • Relative Path: Concatenation of file name starting with the current directory.
  • Change: Changing the current directory to another directory specified by a path name.
  • Create: Creating a new directory and an entry in the current working directory pointing to the new directory.
  • Move: Moving a file or directory from one directory to another.
  • Rename: Changing the name of a file or directory specified by a path name.
  • List: Listing all files and other attributes of files contained in the directory specified by a path name.
  • Find: Finding a file or directory specified by name.
  • File Attributes: Include size, type, location, protection, and use.
  • File Control Block (FCB): A data structure associated with the file name containing all relevant attributes of the file.
  • Disk Block: A fixed sequence of bits on a disk, accessed as a single unit.
  • Contiguous Block Allocation Scheme: Files are mapped into contiguous disk blocks.
  • Linked Block Allocation Scheme: Blocks containing a file may be scattered, with each block pointing to the next.
  • Clustered Block Allocation: Links clusters of contiguous blocks, reducing the number of pointers.
  • Allocation Table: An array where each entry corresponds to a disk block, tracking which blocks belong to a file.
  • Index Block Allocation Scheme: File blocks may reside anywhere on the disk, with an index table tracking blocks.
  • Bitmap: A data structure where each bit represents a disk block (1 for allocated, 0 for free).
  • Opcode: Specifies the type of operation requested.
  • Operands: Describe the parameters of the requested operation.
  • Busy: A register flag indicating whether the device is busy or idle.
  • Status: A register indicating the success or failure of the last I/O operation.
  • Data Buffer: Holds data for transfer between the device and main memory.
  • Programmed I/O: A method where the CPU performs copying of all data between the device controller and memory.
  • Polling: Technique to determine device status by reading a flag set by the device controller.
  • Direct Memory Access (DMA): Allows devices to access main memory directly without CPU involvement.
  • Buffer: A memory area storing data being transferred.
  • Buffer Swapping: Technique using two buffers for overlapping producer and consumer processes.
  • Circular Buffer: A fixed array of buffer slots filled by the producer and emptied by the consumer.
  • Cache: Temporary copy of data stored in reserved memory to improve performance.
  • Spool: A buffer that holds output for devices like printers that cannot handle data streams.
  • Sense Key: Information in the status register indicating an error.
  • Advanced Configuration and Power Interface (ACPI): Firmware managing hardware aspects including power and device information.
  • Mount Table: In-memory data structure containing information about each mounted file system.
  • Streams: A Unix I/O feature allowing dynamic assembly of driver pipelines.
  • Stream Head: Interface between streams and user processes.
  • Driver End: Interface between streams and the device being controlled.
  • Stream Modules: Modules of functionality loadable into a stream.
  • Flow Control: Method to pause the sender of I/O to limit data flow rate.
  • Front-End Processors: Small computers performing tasks in overall systems, managing I/O and offloading the general-purpose CPU.
  • Terminal Concentrator: A type of front-end processor for terminals.
  • I/O Channel: A dedicated special-purpose CPU found in large systems for I/O operations.

Disk Operations


  • Disk Blocks: Fixed-size units of storage on a disk.
  • Seek Time: Time required to move the read/write head to the track containing the desired data.
  • Rotational Delay: Time to wait for the desired data to pass under the read/write head.
  • Data Transfer Time: Time to transfer data to or from the disk.
  • Peak Transfer Time: Rate at which data is streamed to or from the disk.
  • Sustained Data Rate: Rate at which the disk can continuously transfer data, including seek time and overhead.
  • Shortest Seek Time First (SSTF): Scheduling algorithm selecting the request requiring the shortest travel distance.
  • SCAN Scheduling Algorithm: Services requests in one direction; when the outmost request is serviced, the head moves back to the opposite end.

Input/Output Management


  • Programmed I/O: Transfers data one byte at a time between CPU and peripheral device.
  • Direct Memory Access (DMA): Allows devices to transfer large amounts of data directly to/from memory.
  • Scatter-Gather: Input/Output method where multiple sources/destinations are specified in one command.
  • Double Buffering: Technique involving two buffers to decouple producers and consumers.
  • Cycle Stealing: DMA device temporarily uses the bus, preventing the CPU from using it.
  • Direct Virtual Memory Access (DVMA): DMA-like method for virtualized memory access.
  • Interrupt: Signal indicating the need for CPU attention, typically for handling I/O requests.
  • Interrupt Service Routine (ISR): Function executed in response to an interrupt.
  • Non-Maskable Interrupt (NMI): High-priority interrupt that cannot be ignored or masked.
  • Interrupt Vector: Table of pointers to ISR routines for handling various interrupts.

Device Interfaces


  • Device Controller: Hardware component managing communication between the CPU and a peripheral device.
  • Memory-Mapped I/O: Technique where device registers are mapped to memory addresses.
  • Bus: A communication system connecting multiple components, e.g., PCIe.
  • Expansion Bus: Interface providing additional bus lines for connecting more devices.
  • Network Interface Card (NIC): Hardware for network connectivity and communication.

File and Disk Management Tools


  • File Allocation Table (FAT): File system structure managing file and disk space allocation.
  • Error Detection and Correction: Methods for detecting and correcting errors, e.g., parity bits, ECC.
  • File Control Block (FCB): Data structure storing information about a file.
  • Device Status Table: Data structure holding device information and status.

Specialized Concepts


  • Front-End Processor: Computer managing peripheral devices to offload tasks from the main CPU.
  • Terminal Concentrator: Device managing multiple terminals for efficient communication.
  • High-Performance Timers: Specialized timers providing accurate and fast timing functions.
  • Programmable Interval Timer (PIT): Timer allowing programs to set timing intervals.
  • Network Time Protocol (NTP): Protocol for synchronizing clocks over a network.
  • Advanced Configuration and Power Interface (ACPI): Interface for managing hardware configurations and power settings.
